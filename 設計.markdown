```mermaid
sequenceDiagram
    participant front
    participant back
    loop
        front->>back:ゲーム開始のリクエスト
        back->>front:52枚のトランプの状態を初期化し、配布する2枚のカードの情報をレスポンス<br/>このとき、先手がどちらかの情報も加え、後手が最初に賭ける金額の情報も追加
    end

    loop
        front->>back:1人目のフォールド、チェック、レイズ、コールの判断を送信
        back->>front:プレーヤーの判断に従って、次のプレイヤーの選択肢を送信<br/>またはフォールドの場合はテーブル上のチップを相手側のプレイヤーが獲得してゲームを初期化(チップの枚数はそのまま)
        front->>back:二人目の判断を送信
        back->>front:フォールドの場合は初期化、それ以外の場合、テーブルに新たに表向きにして表示するカードの情報をレスポンス
        front->>back:表向きにするカードの情報が空だった場合、ゲーム終了時の処理を依頼
        back->>front:役の強さによって勝敗を判定し、プレーヤーそれぞれの役、勝者、チップ支払い後の状態をレスポンス
        back->>front:もしどちらかのチップが0枚だった場合、ゲーム全体での勝敗が決定したことを合図し、勝者の情報を送信
    end

```
